## Project UI Conventions

- Use AWS icon components for UI visuals instead of emoji when representing AWS services or cloud concepts.
- Prefer the shared component `components/AwsServiceIcon.tsx` for service icons in page-level UI.
- If a required icon is not mapped yet, extend `components/AwsServiceIcon.tsx` first before adding direct icon imports to pages.
- Keep icon style consistent across non-category UI sections (hero, guides, cards, headings).
- For Mermaid architecture diagrams, treat icons as optional enhancement; keep labels explicit so diagrams remain readable without custom icons.

### p5.js canvas visualizations

- Do **not** use emoji as icon labels inside p5 canvas (cross-platform rendering is unreliable).
- Use `p5.loadImage()` to load SVG files from `public/icons/aws/` and draw them with `p5.image()`.
- SVG files in `public/icons/aws/` are generated by `scripts/extract-aws-icons.cjs` from `aws-react-icons`. Run the script when a new icon is needed.
- Store loaded images in `useRef` (e.g. `imgEc2`, `imgIgw`) and call `loadImage` inside the p5 `setup` callback.
- Keep plain-text labels alongside icons so the diagram is readable if images fail to load.

### Layout & styling for p5 elements

- **Corner radius**: Minimize rounding on boxes; use `borderRadius: 0` or small value (≤4px) only when needed.
- **Icon placement**: Position icons at top-left edge with no padding/offset (e.g. icon starts at box top-left corner).
- **Alignment**: When only icon + text present, center-align both horizontally and vertically for balance.
